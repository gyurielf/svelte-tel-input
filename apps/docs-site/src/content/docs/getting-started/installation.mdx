## Prerequisites

- **Svelte 5** (peer dependency)
- **Node.js** >= 22
- **pnpm** >= 10 (or npm / yarn)

## Install

<Tabs>
  <TabItem label="pnpm">
    ```bash
    pnpm add svelte-tel-input
    ```
  </TabItem>
  <TabItem label="npm">
    ```bash
    npm install svelte-tel-input
    ```
  </TabItem>
  <TabItem label="yarn">
    ```bash
    yarn add svelte-tel-input
    ```
  </TabItem>
</Tabs>

## Package exports

| Import path | What you get |
|---|---|
| `svelte-tel-input` | `TelInput` component, `countries` array, `parse()`, `normalizeToE164()`, `pickCountries()`, `inputParser()`, `isSelected()`, `parsePhoneInput()` |
| `svelte-tel-input/types` | All TypeScript types (`CountryCode`, `DetailedValue`, `TelInputOptions`, `ValidationError`, `Props`, …) |
| `svelte-tel-input/validators` | `validateTelInput()` — framework-agnostic validator for Zod / Valibot / Yup |
| `svelte-tel-input/styles/flags.css` | CSS sprite flags for country dropdowns |
| `svelte-tel-input/utils` | Low-level utilities (`clickOutsideAction`, etc.) |
| `svelte-tel-input/assets` | Raw country data |

Re-exports from `libphonenumber-js`: `parsePhoneNumberWithError`, `ParseError`.
