All props listed below are passed directly to the `<TelInput>` component. Props marked **bindable** support Svelte's `bind:` syntax.

## Core props

| Prop | Type | Default | Bindable | Description |
|------|------|---------|----------|-------------|
| `value` | `string` | `''` | Yes | E.164 formatted phone number (e.g. `+36301234567`). This is your source of truth for storage. |
| `country` | `CountryCode \| null \| undefined` | `null` | Yes | ISO 3166 Alpha-2 country code (e.g. `'HU'`, `'US'`). Auto-detected from `value` when not set. |
| `valid` | `boolean` | `true` | Yes | Whether the current input is a valid phone number. |
| `detailedValue` | `DetailedValue \| null` | `null` | Yes | Full parsed result with multiple formats. See [Types](/reference/types/). |
| `el` | `HTMLInputElement \| undefined` | `undefined` | Yes | Reference to the underlying `<input>` element. |
| `options` | `TelInputOptions` | See [Options](/reference/options/) | No | Configuration object for behavior tuning. |

## HTML input props

These are forwarded to the underlying `<input type="tel">` element:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `id` | `string` | Auto-generated | Sets the `id` attribute. Auto-generates a unique ID if not provided. |
| `name` | `string \| null` | `null` | Sets the `name` attribute. |
| `class` | `string` | `''` | CSS class(es) applied to the input. |
| `disabled` | `boolean` | `false` | Disables the input and the parser. |
| `readonly` | `boolean \| null` | `null` | Makes the input read-only. |
| `required` | `boolean \| null` | `null` | Marks the input as required. When set, an empty input is considered invalid. |
| `placeholder` | `string \| null` | `null` | Custom placeholder. Overrides `autoPlaceholder` when set. |
| `size` | `number \| null` | `null` | HTML `size` attribute. |
| `autocomplete` | `string \| null` | `null` | HTML `autocomplete` attribute (e.g. `'tel'`). |

The component also accepts any other standard HTML input attributes via `...rest` spread.

## Bindable props in detail

### `value`

Always stored in [E.164 format](https://en.wikipedia.org/wiki/E.164) — the international standard for phone numbers. Example: `+12154567890`.

```svelte
<script>
  let phoneNumber = $state('');
</script>

<TelInput bind:value={phoneNumber} />
<p>Stored: {phoneNumber}</p>
```

### `country`

Accepts any ISO 3166 Alpha-2 country code. The component auto-detects the country from the value's dial code when possible:

- If `value` is `+44...` and `country` is not set → auto-detects `GB`
- If `country` is explicitly set (e.g. `'US'`), uses that for formatting
- When the user types a number with a different dial code, `country` updates to match

### `valid`

Reflects phone number validity based on `libphonenumber-js` validation. Behavior depends on the `validateOn` option and `required` prop:

- **Empty input + `required` not set** → `valid = true`
- **Empty input + `required` set** → `valid = false`
- **Partial/invalid number** → `valid = false`
- **Complete valid number** → `valid = true`
